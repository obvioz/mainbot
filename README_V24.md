# v24 Decision Engine Rebuild

Главное изменение: бот больше не выдает покупку только потому, что монета сильнее BTC.

Новая логика:
1. Сначала оценивается цена входа: есть ли реальная скидка от 30d high.
2. Затем учитываются сила к BTC, новости, funding/OI, рыночный режим и reliability.
3. Если монета близко к 30d high и уже выросла за 7 дней — это WATCH, даже если она сильная.
4. ACCUMULATION возможен только когда цена стала интересной для нашей стратегии.
5. Для открытых позиций бот отдельно решает: держать, докупать, фиксировать или не докупать.

Новые поля в /scan:
- price EXPENSIVE / NOT_INTERESTING / EARLY_DIP / FAIR / GOOD / GREAT
- action
- price attractiveness внутри карточки

Старый баг с “Ориентир первого входа: 1.8 USDT” исправлен: теперь это “Размер пробного входа”.
