# Crypto Invest Bot v19 — Strategy Lab Integrated

В этой версии Strategy Lab встроен прямо в Telegram-бота.

## Главное

Bybit остается источником истины для портфеля, а Strategy Lab теперь подбирает параметры стратегии и сохраняет их в `data/strategy_params.json`.

После запуска `/lab` бот начинает использовать оптимизированные:

- DCA-лесенки;
- первые TP уровни;
- параметры по каждой монете отдельно.

## Новые команды

```text
/lab
/lab ETH
/lab 2024-01-01 2026-05-30
/lab ETH 2024-01-01 2026-05-30
/strategyparams
```

## Как пользоваться

1. Запусти бота:

```bash
python main.py
```

2. В Telegram нажми `🧪 Lab` или отправь:

```text
/lab 2024-01-01 2026-05-30
```

3. Бот переберет DCA/TP, сохранит лучшие параметры и пришлет CSV-отчет.

4. Проверь активные параметры:

```text
/strategyparams
```

5. После этого `/scan`, DCA и TP подсказки используют сохраненные параметры.

## Важно

Это грубая лаборатория по дневным свечам. Она не учитывает комиссии, спред, внутридневные проколы, funding/news в полной мере. Но она уже помогает выкидывать слабые гипотезы и перестраивать стратегию на основе истории.


## v20 Five-Year Strategy Lab

Strategy Lab теперь по умолчанию тестирует последние 5 лет истории:

```text
/lab
/lab 5y
/lab ETH 5y
/lab 2022-01-01 2026-05-30
/strategyparams
```

После `/lab` лучшие DCA/TP параметры автоматически сохраняются в `data/strategy_params.json` и используются в `/scan`, DCA и TP-подсказках.

Если монета торгуется на Bybit меньше 5 лет, бот использует максимум доступных свечей по этой монете.

## v21 Strategy Robustness Engine

Эта версия добавляет проверку устойчивости стратегии, чтобы не подгонять параметры только под прошлое.

### Новые команды

```text
/robust
/robust 5y
/robust ETH 5y
/robust 2022-01-01 2026-05-30
/robust ETH 2022-01-01 2026-05-30
/reliability
```

### Что делает /robust

- делит историю по годам;
- проверяет параметры DCA/TP на разных рыночных режимах;
- штрафует стратегии с малым числом сделок;
- штрафует минусовые годы;
- делает walk-forward проверку: параметры подбираются на прошлом периоде и проверяются на следующем;
- сохраняет рейтинг надежности в `data/strategy_reliability.json`;
- сохраняет устойчивые параметры в `data/strategy_params.json`.

### Как это влияет на сигналы

После `/robust` бот добавляет в Signal Score историческую устойчивость монеты:

- высокая reliability усиливает BUY-сигнал;
- низкая reliability снижает сигнал и переводит монету в WATCH/NO_SIGNAL;
- `/reliability` показывает рейтинг монет.

### Рекомендуемый порядок

```text
/robust 5y
/reliability
/strategyparams
/scan
```

Важно: это не гарантия прибыли. Это фильтр против переобучения и слабых монет.
